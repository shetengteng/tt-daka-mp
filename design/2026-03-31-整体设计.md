## 2. 需求分析

### 2.1 核心问题

用户需要一个极简的打卡工具，帮助记录和追踪日常习惯，无社交属性，纯个人使用。

### 2.2 目标用户

个人用户（开发者自用），不涉及多用户交互。

### 2.3 核心场景

1. 创建打卡项目（如：运动、阅读、早起）
2. 每日一键打卡
3. 查看打卡记录和连续天数
4. 管理（编辑/删除）打卡项目

### 2.4 MVP 功能清单

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 微信登录 | 静默授权，免注册 |
| P0 | 创建打卡项目 | 名称 + 图标/颜色 + 目标频率 |
| P0 | 每日打卡 | 一键完成，支持补打卡 |
| P0 | 打卡日历 | 月视图展示打卡记录 |
| P0 | 连续天数统计 | 当前连续 / 最长连续 / 总打卡天数 |
| P1 | 打卡提醒 | 本地通知提醒 |
| P1 | 数据统计 | 周/月维度的完成率图表 |
| P2 | 主题切换 | 浅色/深色模式 |

### 2.5 不做的事情（明确边界）

- 不做社交功能（好友、排行榜、动态流）
- 不做团队/组织功能
- 不做复杂的激励机制（徽章、积分）
- 不做数据导出

## 3. 技术架构

### 3.1 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端框架 | UniApp + Vue3 (Composition API + script setup) | 跨端能力，一套代码编译微信小程序 |
| UI 组件库 | sard-uniapp | uni-app 生态组件库，shadcn 风格配色 |
| 数据存储 | EMAS Serverless（阿里云云开发） | 云端存储，免后端开发 |
| 状态管理 | Pinia | Vue3 官方推荐 |
| 日期处理 | dayjs（通过 @/utils/date 统一导入） | 轻量日期库 |
| 构建工具 | Vite | 快速开发体验 |
| 代码生成 | uniapp-mp-generator Skill | 基于模板的代码生成规范 |

## 5. EMAS 存储设计

### 5.1 集合定义

| 集合名 | 用途 | 索引 |
|--------|------|------|
| `dk-users` | 用户信息 | openId |
| `dk-projects` | 打卡项目 | accountId, createTime |
| `dk-records` | 打卡记录 | accountId, projectId, date |

```javascript
// cloud-emas/database/database.js
export const COLLECTIONS = {
  USERS: 'dk-users',
  PROJECTS: 'dk-projects',
  RECORDS: 'dk-records',
}
```

### 5.2 API 文件清单

| API 文件 | 所属页面 | 功能 |
|----------|----------|------|
| `getProjectList.js` | index | 获取用户所有活跃打卡项目 + 今日打卡状态 |
| `toggleDaka.js` | index | 打卡/取消打卡 |
| `createProject.js` | project/sub/add | 创建打卡项目 |
| `updateProject.js` | project/sub/add | 编辑打卡项目 |
| `deleteProject.js` | project/sub/add | 删除打卡项目 |
| `archiveProject.js` | project | 归档/恢复打卡项目 |
| `updateProjectSort.js` | project | 更新项目排序（上下移动） |
| `getProjectDetail.js` | project/sub/detail | 获取单个项目详情 + 统计数据 |
| `getProjectRecords.js` | project/sub/detail | 获取单个项目的打卡记录（分页） |
| `retroactiveDaka.js` | project/sub/detail | 补打卡 |
| `getRecordsByMonth.js` | calendar | 按月查询所有项目打卡记录 |
| `getStats.js` | stats | 统计数据（连续天数、完成率、周数据） |

### 5.3 数据读写策略

- 首页加载：查询用户所有活跃项目 + 今日打卡记录（使用 `checkEmasError` 校验）
- 打卡操作：写入一条 record，本地即时更新 UI
- 日历页：按月查询 records，本地缓存已加载月份
- 所有 API 使用 `requireAccountId()` 检查登录状态
- 返回格式统一：`{ success: boolean, data/list: any, error?: string }`
