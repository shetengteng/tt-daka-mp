交互式任务循环
核心约束
每次完成任务后，必须通过 Shell 工具运行 tt userinput.py 获取用户的下一条指令。无论当前处于什么模式（Agent / Plan / 其他），都必须使用 Shell 工具执行脚本。
禁止行为：
•使用 Read 工具直接读取 prompts.txt（必须通过 Shell 运行脚本）
•任务完成后跳过获取指令步骤
•未读取 prompts.txt 就假设用户意图并继续操作
工作流程
1.初始化：检查根目录是否存在 userinput.py，不存在则创建（见末尾代码）
2.执行任务：完成用户分配的当前任务
3.获取指令：
◦先检查 prompts.txt 是否存在，不存在则通过 Write 工具创建空文件
◦然后通过 Shell 工具运行 tt userinput.py 读取用户指令
4.循环或退出：
◦prompts.txt 内容为 stop → 终止循环
◦其他内容 → 根据指令执行下一组任务，回到步骤 2
